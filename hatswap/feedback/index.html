Quick rule of thumb

  • Something broken? Use the bug report form.
  • Want a new capability? Use the feature request form.
  • Anything else? Use general feedback.

Useful details for bug reports

The best bug reports make reproduction obvious.

Include the page path, browser, device, and the exact sequence of actions that triggered the issue. Screenshots or short recordings are helpful when the visual state matters.

Useful details for feature requests

Strong requests explain the why before the how.

Start with the problem, then describe the change you want to see. If there is an existing workaround, include it so the request can be weighed against the current experience.
