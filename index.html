Feedback first

The design keeps support routes visible and consistent so visitors can move from page to issue form without guessing where to go next.

Project focus

HatSwap is the main destination.

The landing page now acts as a clean entry point to the project-specific feedback flow. That keeps the main navigation light while still making the next action obvious.

Design approach

Readable, responsive, and low-friction.

The visual system uses a warm paper-like background, layered cards, and a restrained color palette to give the site a polished feel without adding unnecessary complexity.

Feedback flow

Three paths, one clear handoff.

Each issue form gathers the details needed to act quickly. Users can report a bug, ask for a feature, or leave general feedback without sorting through a long blank issue.

Bug reports

  • Reproduction steps
  • Expected versus actual behavior
  • Environment details and screenshots

Feature requests

  • Problem statement
  • Suggested solution
  • Alternatives or edge cases

Direct links to issue forms

The Hatswap page links straight into GitHub's issue templates so visitors can submit the right kind of feedback with fewer steps.

  • Bug report: bug_report.yml
  • Feature request: feature_request.yml
  • General feedback: general_feedback.yml

About this site

Built for GitHub Pages, with room to grow.

The page structure is intentionally small so future project pages can inherit the same visual language without introducing a full framework or a complicated build chain.
